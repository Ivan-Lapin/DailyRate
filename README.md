# Currency-service

Сервис микросервисного приложения для получения и предоставления курса валют.  

## Описание

Данный сервис запрашивает курсы валют с внешнего API, кеширует и хранит их в базе PostgreSQL. Сервис предоставляет API для получения текущих курсов, исторических значений и курсов за конкретные даты через gRPC и REST интерфейсы.  

Встроена поддержка надежных повторных попыток получения данных с использованием алгоритма backoff, планировщик заданий (Cron) для автоматического обновления курсов, а также мониторинг состояния и метрик через Prometheus и Grafana.  

## Технологии и инструменты

- Язык программирования: Go  
- API: gRPC и REST  
- СУБД: PostgreSQL  
- Логирование: zap.Logger  
- Мониторинг: Prometheus, Grafana  
- Документация: Swagger  
- Контейнеризация: Docker, Docker-compose  
- Планировщик задач: Cron  
- Retry-логика с backoff  
- Graceful shutdown  

## Структура проекта

- `cmd/cron` — пакет с планировщиком для запуска periodic fetch запросов к API  
- `cmd/currency` — основной сервис с реализацией gRPC/REST серверов  
- `internal/clients/currency` — клиент для внешнего API  
- `internal/config` — конфигурация сервиса  
- `internal/handler` — обработчики gRPC запросов и middleware, например JWT авторизация  
- `internal/repository` — уровень работы с БД  
- `internal/service` — бизнес-логика получения курсов и сервисные методы  
- `internal/storage` — низкоуровневый интерфейс для работы с PostgreSQL, включая метрики Prometheus  
- `proto/currency` — protobuf описание API и сгенерированные gRPC файлы  
- `monitoring/prometheus` — конфигурация мониторинга для Prometheus
